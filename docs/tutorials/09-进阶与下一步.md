# 文章 9 — 进阶与下一步

学习目标：为深入主题提供入口，并给出持续学习与团队推广的路线建议。

## 9.1 进阶专题导览

- EPON 与点到多点：OLT/ONU 的 SC/SA 设计、SCB 广播安全、密钥分发策略与撤销流程。
- 多接入 LAN 与虚拟 SecY：通过 SCI/虚拟端口复用共享媒质，避免多桥互联导致拓扑漂移。
- EDE 家族与运营商场景：EDE-M/CS/CC/SS 的端口角色、组播过滤与跨域互联细节。
- LLDP 与发现：在 Uncontrolled/Controlled 之间的权衡，如何在安全与可见性间达成平衡。
- 零信任与身份驱动：结合 802.1X 与策略编排，实现“身份到链路”的一致安全。

## 9.2 实操项目建议

- 自动化实验室：使用 Linux macsec + wpa_supplicant 搭建 CI 场景，覆盖重键、乱序、窗口与故障注入。
- 指标平台：把 SecY 计数器/事件汇聚至你的可观测性系统，建立看板与告警规则。
- 知识库建设：整理 MACsec 与 802.1X/802.1Q/IPsec/VXLAN 的对比表，沉淀组织内最佳实践。

## 9.3 团队推广与培训路径

- 一级（网络工程师）：概念/实验/排错三部曲；能独立部署与回退。
- 二级（平台/安全）：套件选型、密钥策略、运维规范、自动化工具链。
- 三级（架构/合规）：系统边界治理、跨域方案、合规与审计流程建设。

## 9.4 持续改进清单

- 图表与示意：为常用时序与字段画出标准化图例，便于跨团队沟通与培训。
- 测试深度：补充极端帧长/队列策略/混合媒体场景的压力测试。
- 生态跟踪：关注 IEEE 802.1 工作组、厂商互通测试（Plugfest）与开源实现演进。

## 9.5 结语

至此，你已完成从动机→概念→机制→实操→生产→运营→合规→进阶的完整学习路径。建议将本系列纳入团队的入职与值班必读清单，并以此为基线开展自动化工程与平台化治理。
